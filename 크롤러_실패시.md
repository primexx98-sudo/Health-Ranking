# 올리브영 크롤러 실패 시 — Claude 처리 매뉴얼

카카오톡으로 "올리브영 수집 실패" 알림을 받았을 때, 이 파일 + 랭킹 페이지 캡쳐 이미지를
Claude Code에 함께 전달하면 아래 절차를 그대로 수행하면 됩니다.

**프로젝트 경로**: `C:\Users\thefuture_brand\Desktop\업무\시장조사`

## 배경 (왜 이 절차가 필요한가)

올리브영은 GitHub Actions(Azure 데이터센터 IP)에서 Cloudflare Turnstile 인터랙티브
챌린지("사람인지 확인하십시오")로 막힌다. stealth 제거, 체크박스 자동 클릭, 헤더 조합,
IP 랜덤 재시도까지 모두 시도했으나 코드 레벨로는 우회 불가로 결론남(설계서.md 6장).
반면 사용자 가정용 IP는 항상 정상 통과하므로, 실패 시에만 사용자가 직접 캡쳐해
Claude에게 전달하는 방식으로 보완한다. 카카오·다이소는 이 문제와 무관하게 항상
자동 정상 수집된다.

## Claude가 할 일 (순서대로)

1. **날짜 확인**: 캡쳐가 오늘 날짜 기준인지 사용자에게 확인(다른 언급 없으면 오늘).
2. **`data/daily/YYYY-MM/YYYY-MM-DD.xlsx` 존재 확인**. 없으면 그날 카카오·다이소 자동
   수집 자체가 아직 안 됐거나 그것도 실패한 것 — 이 경우 `git pull`로 최신 상태부터
   받고, 그래도 없으면 `python main.py`로 먼저 그날 수집을 실행해야 함(3개 플랫폼
   전부 실패했을 가능성 있으니 이 경우는 사용자에게 상황 설명).
3. **캡쳐 이미지를 Read 도구로 읽어 TOP10 추출**: 순위, 브랜드, 상품명, 가격(할인가
   있으면 할인가, 없으면 정가). 상품URL은 캡쳐에 안 보이므로 생략(빈 값).
   - 10개 미만만 보이면(스크롤 잘림 등) 사용자에게 나머지 캡쳐를 추가 요청.
4. **items.json 작성**: 스크래치패드에 아래 형식으로 저장.
   ```json
   [
     {"순위": 1, "브랜드": "...", "상품명": "...", "가격": "8500"},
     ...
   ]
   ```
5. **주입 스크립트 실행** (프로젝트 루트에서):
   ```bash
   python manual_oliveyoung.py <YYYY-MM-DD> <items.json 경로>
   ```
   → `data/daily/YYYY-MM/YYYY-MM-DD.xlsx`의 "올리브영" 시트가 추가/교체되고
   "카테고리통계" 시트가 재계산됨.
6. **결과 검증**: 해당 xlsx를 pandas로 열어 "올리브영" 시트 행 수(=캡쳐로 준 개수)와
   "카테고리통계" 합계(카카오+다이소+올리브영 합)가 맞는지 확인.
7. **git 반영**: `git pull --rebase` → `git add data/daily/...` →
   `git commit -m "..."` → `git push` (Actions 자동 커밋과 충돌 시 `git pull --rebase`로
   먼저 최신화, 데이터 파일 하나뿐이라 충돌 시 로컬 버전이 최신이므로 그대로 채택).
8. **사용자에게 결과 요약 보고**: 반영된 개수, 카테고리 분류 결과, push 완료 여부.
