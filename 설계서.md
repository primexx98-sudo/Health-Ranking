# 건강기능식품 판매순위 크롤러 설계서

작성일: 2026-06-25

---

## 1. 프로젝트 개요

3개 플랫폼(올리브영, 다이소몰, 카카오 선물하기)의 건강기능식품 TOP10 판매순위를
매일 자동 수집하여 일별 엑셀로 저장하고, 월말에 평균 순위 기준 월간 TOP10을 집계한다.

|--------|-----|-----------|
| 카카오 선물하기 | https://gift.kakao.com/category/5/subcategory/99 | 1~10위 |
| 다이소몰 | https://www.daisomall.co.kr/ds/exhCtgr/C208/CTGR_00022/CTGR_01020 | 1~10위 |
| 올리브영 | https://www.oliveyoung.co.kr/store/display/getCategoryShop.do?dispCatNo=10000020001 | 1~10위 |

## 4. 자동화 방식

- **플랫폼:** GitHub Actions (클라우드, PC 비활성 상태에도 실행)
- **실행 주기:** 매일 오전 9시 (KST = UTC 00:00)
- **실패 알림:** GitHub 이메일 알림 (Actions 실패 시 자동 발송)

## 6. 엑셀 파일 구조

### 일별 파일 (`data/daily/YYYY-MM-DD.xlsx`)
- 시트 3개: 카카오선물하기 / 다이소몰 / 올리브영
- 각 시트 컬럼: `카테고리 | 순위 | 상품명 | 브랜드 | 가격 | 상품URL`

### 월별 파일 (`data/monthly/YYYY-MM_월별취합.xlsx`)
- 시트 3개: 카카오선물하기 / 다이소몰 / 올리브영
- 각 시트 컬럼: `순위(월평균기준) | 상품명 | 브랜드 | 평균가격 | 평균순위 | 등장횟수`
- 정렬 기준: 평균순위 오름차순 → TOP10만 표시

---

## 7. 월별 집계 로직

```
월평균순위 = SUM(해당월 일별순위) / 등장일수
등장횟수 = 해당월에 TOP10 안에 든 날 수
최종 월간 TOP10 = 평균순위 기준 상위 10개 상품
```

> 순위에 없던 날은 제외하고 평균 계산 (결측일 순위를 11로 처리하지 않음)

## 9. GitHub Actions 워크플로

### daily_crawl.yml
```yaml
name: Daily Crawl
on:
  schedule:
    - cron: '0 0 * * *'   # UTC 00:00 = KST 09:00
  workflow_dispatch:        # 수동 실행 가능

jobs:
  crawl:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - run: pip install -r requirements.txt
      - run: playwright install chromium
      - run: python main.py
      - name: Commit results
        run: |
          git config user.name "github-actions"
          git config user.email "actions@github.com"
          git add data/daily/
          git commit -m "data: $(date '+%Y-%m-%d') 일별 수집" || echo "변경없음"
          git push
```

### monthly_aggregate.yml
```yaml
name: Monthly Aggregate
on:
  schedule:
    - cron: '0 0 1 * *'   # 매월 1일 UTC 00:00 = KST 09:00
  workflow_dispatch:

jobs:
  aggregate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - run: pip install -r requirements.txt
      - run: python monthly_aggregate.py
      - name: Commit results
        run: |
          git config user.name "github-actions"
          git config user.email "actions@github.com"
          git add data/monthly/
          git commit -m "data: $(date '+%Y-%m') 월별 취합"
          git push
```

## 10. GitHub 셋업 절차

1. github.com → New repository → Private → 리포 이름 입력 (예: `health-ranking`)
2. 로컬 PC에 git clone
3. 코드 작성 후 push
4. Settings → Actions → General → Workflow permissions → **Read and write permissions** 체크
5. 첫 번째 수동 실행: Actions 탭 → Daily Crawl → Run workflow

---

## 11. 유지보수 포인트

- 셀렉터 변경 시 `crawlers/config.py`만 수정
- 플랫폼 URL 변경 시 동일하게 `config.py` 수정
- GitHub Actions 무료 플랜: 월 2,000분 제공 (이 프로젝트 예상 사용량 ~60분/월)
