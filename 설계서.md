## 6. 크롤러 방식 (플랫폼별)

| 플랫폼 | 라이브러리 | 추출 방식 | Cloudflare | 수집 수 |
|--------|-----------|-----------|-----------|---------|
| 카카오 | Playwright (headless) | `page.evaluate()` JS — React SPA 대응 | 없음 | 카테고리당 10개 × 2 = 최대 20개 |
| 다이소 | Playwright (headless) | `page.evaluate()` JS — "실시간 랭킹" 텍스트로 섹션 탐색 | 없음 | 10개 |
| 올리브영 | **curl_cffi** | regex — `tx_brand`, `tx_name`, `tx_cur`, `t_number` 파싱 | **IP 차단 → TLS 모방으로 우회** | 10개 |

### 올리브영 Cloudflare 우회 원리
- 일반 `requests` → **403** (Azure IP 포함 클라우드 IP 전체 차단)
- `curl_cffi` + `impersonate="chrome124"` → **200 성공**
- Chrome의 TLS 핑거프린트(JA3/JA4)를 완벽히 모방 → 봇 감지 통과
- GitHub Actions에서도 동일하게 작동 → **PC 없이 완전 자동화**

## 8. 자동화 (GitHub Actions)

| 워크플로 | 트리거 | 동작 |
|---------|--------|------|
| `daily_crawl.yml` | 매일 UTC 00:00 (= KST 09:00) + 수동 | 3개 플랫폼 수집 → xlsx 저장 → git push |
| `monthly_aggregate.yml` | 매월 1일 UTC 00:00 + 수동 | 전월 daily xlsx 집계 → 월별 xlsx 저장 → git push |

**실행 환경:** `ubuntu-22.04`
> ubuntu-latest(=24.04)는 Playwright libasound2 호환 문제로 반드시 22.04 고정

**수동 실행:**
1. https://github.com/primexx98-sudo/Health-Ranking/actions 접속
2. 왼쪽 워크플로 선택 → **Run workflow** 클릭

**실패 확인:**
- Actions 로그에서 오류 메시지 확인
- `screenshots/` 폴더가 Artifact로 업로드됨 (실행 결과 페이지 하단 다운로드)

## 10. 엑셀 파일 구조

### 일별 `data/daily/YYYY-MM-DD.xlsx`
- 시트 3개: **카카오선물하기** / **다이소몰** / **올리브영**
- 카카오 시트: 건강식품·영양제(1~10위) + 다이어트·이너뷰티(1~10위) = 최대 20행

### 월별 `data/monthly/YYYY-MM_월별취합.xlsx`
- 시트 3개: **카카오선물하기** / **다이소몰** / **올리브영**
- 컬럼: `순위(월평균) | 상품명 | 브랜드 | 평균가격 | 평균순위 | 등장횟수`
- 정렬: 평균순위 오름차순 TOP 10만 표시

### 월별 집계 로직
```
월평균순위 = SUM(해당월 일별순위) / 등장일수
등장횟수   = 해당월에 TOP10 안에 든 날 수
월간TOP10  = 평균순위 기준 상위 10개
```
순위에 없던 날은 제외 (결측일을 11위로 처리하지 않음)
