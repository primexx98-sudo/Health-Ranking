## 6. 크롤러 방식

| 플랫폼 | 라이브러리 | 핵심 셀렉터 | Cloudflare | 수집 수 |
|--------|-----------|-----------|-----------|---------|
| 카카오 | Playwright (headless) | `gc-product` 웹컴포넌트, `span.txt_prdbrand`, `strong.txt_prdname` (`.area_ad`/`.txt_ad` 광고 제외) | 없음 | 카테고리당 10개×2=최대 20개 |
| 다이소 | Playwright (headless) | `a.detail-link[href]`(pdNo 기준) + 상세페이지 `a.brand-area .detail-title` 브랜드 보강 | 없음 | 10개 |
| 올리브영 | curl_cffi + BeautifulSoup | `ul.cate_prd_list > li`, `span.tx_brand`, `p.tx_name` | **IP 차단 → curl_cffi TLS 모방(impersonate)으로 우회** | 10개 |

- 다이소: 목록 카드에 브랜드가 없어 TOP10 상세페이지를 추가 방문해 브랜드 추출 (다소 느림). `swiper-slide-duplicate` 제외, 부족하면 `.swiper-button-next` 클릭 fallback.
- 올리브영: 일반 `requests`는 403. `curl_cffi`가 Chrome TLS 핑거프린트를 모방해 봇 감지를 통과. 버전 만료 시 유지보수 가이드(9장) 참고.

## 8. 자동화 (GitHub Actions)

| 워크플로 | 트리거 | 동작 |
|---------|--------|------|
| `daily_crawl.yml` | 매일 UTC 00:00(=KST 09:00) + 수동 | 3개 플랫폼 수집 → 카테고리 분류 → xlsx 저장 → git push |
| `monthly_aggregate.yml` | 매월 1일 UTC 00:00 + 수동 | 전월 daily xlsx 집계 → 카테고리 통계 → 월별 xlsx 저장 → git push |

- **실행 환경**: `ubuntu-22.04` 고정 (ubuntu-latest=24.04는 Playwright libasound2 호환 문제 발생)
- **필수 설정**: 두 워크플로 모두 `permissions: contents: write` + commit 후 `git pull --rebase` 포함 (누락 시 각각 권한 오류/push 충돌로 데이터 유실)
- **수동 실행**: https://github.com/primexx98-sudo/Health-Ranking/actions → 워크플로 선택 → Run workflow
- **실패 확인**: Actions 로그 확인, `screenshots/`가 Artifact로 업로드됨

## 10. 엑셀 파일 구조

### 일별 `data/daily/YYYY-MM/YYYY-MM-DD.xlsx` — 시트 4개

`main.py`/`app.py`는 저장 시 오늘 날짜의 연-월(`today[:7]`)로 폴더 자동 생성. `monthly_aggregate.py`는 집계 대상 월 폴더에서만 탐색.

| 시트명 | 내용 |
|--------|------|
| 카카오선물하기 | 건강식품·영양제(1~10위) + 다이어트·이너뷰티(1~10위) = 최대 20행 |
| 다이소몰 / 올리브영 | TOP10 |
| 카테고리통계 | 3개 플랫폼 합산 카테고리별 상품수 + 비율(%), 플랫폼별 내역 |

### 월별 `data/monthly/YYYY-MM_월별취합.xlsx` — 시트 4개

| 시트명 | 내용 |
|--------|------|
| 카카오선물하기 / 다이소몰 / 올리브영 | 월평균순위 TOP20 |
| 카테고리통계 | 월간 TOP20 기준 카테고리 비율 |

플랫폼 시트 컬럼: `순위(월평균) | 카테고리 | 상품명 | 브랜드 | 평균가격 | 평균순위 | 등장횟수`

### 월별 집계 로직 (`monthly_aggregate.py`)

```
최소등장   = min(3, 실제수집일수)
           → 필터 후 대상이 20개 미만이면 최소등장을 2회, 1회로 순차 완화해 재필터링
일별점수   = (11 - 순위) × 0.5              # 1위=5점 … 10위=0.5점
평균점수   = SUM(일별점수) / 등장횟수
등장률     = 등장횟수 / 실제수집일수          # 달력 일수가 아닌 실제 파일 수 기준
월간점수   = 평균점수×0.7 + (등장률×5)×0.3   # 순위 70% + 꾸준함 30%
월간TOP20  = (등장횟수 ≥ 최소등장, 완화 적용 후) 대상 중 월간점수 기준 상위 20개 (표시는 평균순위)
```

- 등장횟수가 적어도(3회 미만) 평균순위만 좋으면 TOP20에 오르는 것을 막기 위한 필터. 단, 표본이 적은 플랫폼(예: 다이소몰)에서 3회 기준으로는 20개가 안 채워지면 2회→1회로 완화해 최종 20개를 채움. (플랫폼 일별 수집이 10개뿐이면 실제로는 20개 미만으로 채워질 수 있음)
- 결정 배경·시행착오는 **기록.md** 참고.

## 12. 네이버 DataLab 트렌드 조사 (`run_naver_trend.py`)

네이버 DataLab 공식 API로 건강기능식품·일반식품 검색어/쇼핑 클릭 트렌드를 크롤링 없이 수집 (최근 1년 월별 지수).

**사전 준비**: developers.naver.com에서 애플리케이션 등록 (데이터랩 검색어 트렌드 + 쇼핑 인사이트 API 선택) → 발급된 Client ID/Secret을 `.env`에 입력 (`.env.example` 참고).

**실행**: `python run_naver_trend.py`

| 수집 항목 | 저장 경로 |
|-----------|-----------|
| 건강기능식품/일반식품 검색어 트렌드 (각 10종 키워드) | `data/naver/search/*_YYYYMMDD.csv` |
| 분야별 + 키워드별 쇼핑 인사이트 클릭 트렌드 | `data/naver/shopping/*_YYYYMMDD.csv` |

**API 제한**: 검색어 트렌드 요청당 5개 그룹×20개 키워드, 쇼핑 인사이트 요청당 5개 키워드, 조회기간 최대 2년, 일 1,000회.
